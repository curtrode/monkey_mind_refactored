Monkey Mind

Too many tabs open — from stochastic duet to improvisation
Curt Rode

About This Work-in-Progress Excerpt

Monkey Mind is an evolving electronic literature project that begins as a stochastic duet and moves toward improvisational response. In its current two-voice form, the piece presents parallel streams of generated text assembled from overlapping but distinct word pools. Each voice combines relational subjects ("you are my," "I am not your," "we were our") with abstract and pastoral predicates ("silence," "horses," "yellow bird," "darkness"), producing a continuous texture of intimacy, negation, and silence.

The project runs in a browser with a terminal-style interface and can be experienced live in two forms: the duet at curtrode.github.io/monkey_mind and the improvisation prototype at curtrode.github.io/monkey_mind/openAI-improv.html. In the duet, the voices begin muted; the viewer chooses when — and whether — to listen. Text-to-speech narration uses both synthetic machine and AI-generated voices; in AI mode, the two voices are spatialized in stereo, one left and one right.

The work begins in Nick Montfort's Memory Slam lineage — a collection of stochastic text engines reimplementing classics like Christopher Strachey's Love Letters (1952) and Alison Knowles & James Tenney's A House of Dust (1967). Where those foundational works operate as poetic monologues, Monkey Mind starts by carrying stochastic generation into dialogue and now extends that lineage further through a transition from combinatory generation to improvisational response.

What follows documents the stochastic engine and duet structure from which the larger project evolves: the generation logic, the word pools that constitute its vocabulary, and a sample of the language it produces. The full project, including interface, speech logic, and the live improvisation prototype, is available at github.com/curtrode/monkey_mind. In computational writing, the code is the writing — the literary decisions are embedded in the choice of words, the sentence templates, the timing, the transitions, and the silences.

'use strict';

const MonkeyMind = {
  init(config) {
    const {
      subjects,
      predicates,
      conjunctions,
      operators,
      interval,
      maxVisible = 8,
      endearments = null,
      interjections = null,
      id = null,
    } = config;

    let t = 0;

    const randRange = (maximum) =>
      Math.floor(Math.random() * (maximum + 1));

    const choose = (array) =>
      array[randRange(array.length - 1)];
choose() selects a random element from any array — the single point of stochastic decision-making in the entire engine. 
    const phrase = () => {
      let text = choose(operators);
      if (endearments) {
        text += choose(endearments);
      }
      text += ' ' + choose(subjects);
      if (text === 'A EYE') {
        text = 'AN EYE';
      }
      return text + ' ';
    };
A phrase is: an operator (currently empty) + an optional endearment + a subject. The 'A EYE' → 'AN EYE' correction is a vestige of the original Montfort engine, preserved here. 
    const litany = () => {
      const main = document.getElementById('main');
      if (maxVisible > t) {
        t += 1;
      } else {
        main.removeChild(main.firstChild);
      }
      const last = document.createElement('div');
      let text = phrase() + choose(predicates);

      if (interjections) {
        text += choose(interjections);
      }
      text += choose(conjunctions);
      text += phrase() + choose(predicates) + ' ';
      last.innerHTML = text;
      main.appendChild(last);

      // Send text to parent frame for speech
      if (window.parent !== window && id) {
        window.parent.postMessage(
          { type: 'monkeyMindText', text: text, source: id }, '*'
        );
      }
    };
Each line of output follows the pattern: phrase + predicate + conjunction + phrase + predicate. This is the sentence template — a two-clause structure joined by a conjunction, directly descended from the original stochastic text engines. The postMessage call sends the generated text to the parent frame, where it can be spoken aloud. 
    let currentInterval = interval;
    let timer = null;

    const schedule = () => {
      timer = setTimeout(() => {
        litany();
        schedule();
      }, currentInterval);
    };

    // Listen for interval changes from parent
    window.addEventListener('message', (e) => {
      if (e.data && e.data.type === 'setInterval') {
        currentInterval = e.data.interval;
        clearTimeout(timer);
        schedule();
      }
    });

    litany();
    schedule();
  }
};
Each voice runs on its own timer. "Him" defaults to 3500ms; "her" to 2000ms — she generates nearly twice as fast. The viewer can adjust both speeds. This asymmetry creates the overlapping, unsynchronized rhythm that gives the piece its texture.

The Word Pools

Each voice is initialized with its own arrays of subjects, predicates, and conjunctions. The two voices share most of their vocabulary but diverge in small, deliberate ways — giving each a subtle personality within the shared language of the relationship. Words highlighted in yellow are unique to that voice. Empty strings appear intentionally — when selected, they produce gaps, silence, lines that trail into nothing.

Roughly one-third of subject entries are empty — silence as vocabulary. The relational pronouns cycle through tenses and possessives: present ("you are"), past ("she was"), negated ("I am not"). The parenthetical interjections — whispered asides that break the formal sentence pattern — are shared. Both voices have the same capacity for direct address; neither has a monopoly on vulnerability.

Improvisation

The improvisation prototype stages the transition that the stochastic engine cannot make on its own. A single voice runs the same combinatory grammar shown above, accumulating approximately twenty lines of stochastic output. That accumulated language is then fed as context to an LLM (GPT-4o), which takes over generation in three phases. The author's role shifts from building the grammar to directing the model — shaping each phase through system prompts that function as compositional instructions.

The System Prompts

Each phase is defined by a prompt that tells the LLM how to continue. These are the key directorial passages:

Echo
Continue generating lines in EXACTLY this style. Stay very close to the original grammar patterns. Use the same word pools, the same two-clause sentence structure, the same parenthetical asides. You are still the stochastic engine, just continuing.
Bend
The rigid two-clause template is loosening. You may now: let a line be a single clause or fragment… ask a question… introduce new words that feel like they belong in the emotional world of the piece… let a parenthetical aside become the whole line… reference or transform a phrase from earlier in your output. Something like intention is emerging.
Drift
You have been listening to yourself. Now you are improvising freely — like a jazz soloist who heard the head and is now soloing over the changes. The original vocabulary, rhythms, and obsessions are still in your blood, but you are going somewhere new. You are finding language for things the stochastic engine could not say.
Each prompt receives the full context of everything the voice has said so far. The LLM does not start fresh — it inherits its own history, making each performance cumulative and unrepeatable. Temperature increases across phases (0.7 → 0.9 → 1.0), loosening the model's adherence to pattern.

The voice itself shifts with the language. During the stochastic phase, a synthetic machine voice speaks the lines. As the LLM takes over, an AI-generated voice replaces it. The transition is not just visible on screen but audible — mechanical speech giving way to something that sounds like a person trying to reach someone.
